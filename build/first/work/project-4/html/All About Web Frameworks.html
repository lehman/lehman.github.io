All About Web Frameworks

What is a web framework for?

Web frameworks were designed to make building a website faster and easier. They provide methods of streamlining the mundane and common tasks in web development. One of the main ways they do this is by promoting the reuse of code. Web frameworks can be made to work in ways that are better for certain end results, but they overarchingly designed to support development of dynamic websites, web applications, and web services.

What is a web framework?

A web framework is a collection of tools made to meet what are known as the common CRUD cases:
  • Create
  • Read
  • Update
  • Delete
These are the main functionalities that websites have to interact with their databases and user input/output.

What do web frameworks provide?

The way frameworks work are through libraries which provide certain functionalities. Some of these functionalities are:

  • accessing a database
  • managing sessions and cookies
  • creating templates to display your HTML
  • promoting the reuse of code
  • autheticating users, building an admin backend, all with a few lines of code
  • allowing developers to achieve more, with less coding

Types of Frameworks (How They're Structured)

There are four different types of frameworks. Frameworks are categorized within these types based on how they organize data.

  • Model-View-Controller (MVC)
    • separate data model with business rules from the user interface
    • modularizes code
    • promotes code reuse
    • allows multiple interfaces to be applied
  • Push-Based vs. Pull-Based
    • Push Based: do processing then push data to view layer
    • Pull Based: start with view layer, pull results from multiple controllers when needed; multiple controllers can be involved with a single view
  • Three-Tier Organization
    • structured around three physical tiers:
      • client
      • application
      • database
  • Content Management Systems
    • ex. Wordpress
